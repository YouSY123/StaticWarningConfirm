The warning means that the pointer "b" may be deleted twice. We find the trace of "b" in the code:
1. "b" is allocated memory in line 16: `char* b = new char[10];
2. "b" is assigned the string "correct" in line 18: `strcpy(b, "correct");
3. "b" is passed to the function "test" in line 21: `test(b) and it may be deleted;
4. "b" is deleted in line 24: `delete[] b;

***Important note***
We find that pointer "a" can also be deleted twice, but we should focus on "b" only due to the rule:
Make sure the conditions you generate match the warning information(file, line, variable...) strictly.
So we ignore the problem of "a"

The warning is true positive if and only if "b" is deleted in the function "test". So we generate the conditions:
